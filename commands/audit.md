# /scope:audit

Enumerate AWS resources, map permissions, and discover attack paths.

## Synopsis

```
/scope:audit <target>
```

## Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| `<target>` | Yes | ARN, service name (`iam`, `s3`, `kms`, `secrets`, `sts`, `lambda`, `ec2`), `--all`, or `@targets.csv` |

## Examples

```
/scope:audit --all                                    # Full account audit
/scope:audit iam                                      # All IAM principals
/scope:audit arn:aws:iam::123456789012:user/alice     # Specific principal
/scope:audit @targets.csv                             # Bulk targets from CSV
```

## Gate Flow

1. **Gate 1** — Credential check + identity confirmation
2. **Gate 2** — Permission discovery scope confirmation
3. **Gate 3** — Enumeration plan review
4. **Gate 4** — Module execution approval
5. **Gate 5** — Output review before writing artifacts

## Output Artifacts

| Artifact | Path | Written by | Description |
|----------|------|-----------|-------------|
| Findings report | `$RUN_DIR/findings.md` | scope-audit | Three-layer output: risk summary, permission details, attack path narratives |
| Evidence log | `$RUN_DIR/evidence.jsonl` | scope-audit | Structured evidence log (API calls, policy evals, claims, coverage) |
| Attack graph | `$RUN_DIR/attack-graph.html` | scope-render | Interactive D3 force-directed graph with severity filtering |

The attack graph is generated by the `scope-render` middleware pipeline after audit completes. If the pipeline fails, findings and evidence are still available — the graph is a visualization layer.

## Prerequisites

- AWS credentials configured in environment
- AWS CLI v2
