# /scope:remediate

Generate enterprise-scale SCPs, security controls, and detection suggestions from audit findings.

## Synopsis

```
/scope:remediate
```

## Arguments

None. Reads all prior audit runs from `./audit/`.

## Examples

```
/scope:remediate    # Analyze all audit runs and generate remediation plan
```

## Gate Flow

1. **Gate 1** — Intake summary: audit runs parsed, attack paths aggregated, systemic vs one-off classification
2. **Gate 2** — SCP/RCP policy proposals reviewed before writing JSON files
3. **Gate 3** — Security control recommendations reviewed
4. **Gate 4** — SPL detection suggestions reviewed
5. **Gate 5** — Output documents reviewed before writing

## Output Artifacts

| Artifact | Path | Written by | Description |
|----------|------|-----------|-------------|
| Executive summary | `$RUN_DIR/executive-summary.md` | scope-remediate | Leadership risk scorecard + top quick wins |
| Technical remediation | `$RUN_DIR/technical-remediation.md` | scope-remediate | Full engineer-facing plan with SCP/RCP, controls, detections |
| SCP policies | `$RUN_DIR/policies/scp-*.json` | scope-remediate | Deployable SCP JSON files |
| RCP policies | `$RUN_DIR/policies/rcp-*.json` | scope-remediate | Deployable RCP JSON files |
| Evidence log | `$RUN_DIR/evidence.jsonl` | scope-remediate | Structured evidence log (API calls, claims, coverage) |
| Dashboard | `$RUN_DIR/dashboard.html` | scope-render | Interactive HTML dashboard — risk matrix, policy coverage, MITRE heatmap |

The dashboard is generated by the `scope-render` middleware pipeline after remediate completes. If the pipeline fails, all other artifacts are still available.

## Prerequisites

- At least one completed audit run in `./audit/`
