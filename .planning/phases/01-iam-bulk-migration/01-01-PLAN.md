  <files>bin/validate-enum-output.js</files>
  <action>
Create `bin/validate-enum-output.js` as a Node.js script (stdlib only, no npm dependencies) following the same pattern as the existing `bin/generate-report.js`.

The validator must:
1. Accept a single CLI argument: path to a JSON file (`process.argv[2]`)
2. Print usage and exit 1 if no argument provided
3. Read and parse the file — exit 1 with `[FAIL] Invalid JSON: {message}` on parse error
4. Validate envelope required fields: `module`, `account_id`, `region`, `timestamp`, `status`, `findings`
5. Validate `module` is one of the 12 known modules: iam, sts, s3, kms, secrets, lambda, ec2, rds, sns, sqs, apigateway, codebuild
6. Validate `account_id` matches pattern `^\d{12}$`
7. Validate `status` is one of: complete, partial, error
8. Validate `findings` is an array
9. For each finding in `findings[]`, validate required fields: `resource_type`, `resource_id`, `arn`, `region`, `findings` (inner findings array)
10. On success: print `[OK] {filepath} passes validation ({N} findings checked)` and exit 0
11. On failure: print each `[FAIL] ...` error and exit 1 with count

Do NOT use ajv or any JSON Schema library — plain Node.js field checks per the CONTEXT.md decision.

Make the file executable with `chmod +x`.
